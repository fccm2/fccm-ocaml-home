Of course after we add new letters, we have to recompile:

$ ocamlc -c tr_alph.mli
$ ocamlc -c tr_alph.ml

In the previous version of the tutorial, there was no .mli yet.

Of course it's better if we create one, because we can clearly see the functions that we can use, their interface, and we can add some documentation, to explain how to use every element of the module.

After recompil'd, the module is the .cmo file, which is immediately usable like this:

$ \ocaml -I . tr_alph.cmo

An interface always has to be compil'd before the implementation. Compiling the .mli file, produces a .cmi file.

The option -I . tells our interactive loop, that our module is to be search'd in the directory where we are.

An interface always has to be compil'd before the implementation, in order to guaranty that the implementation will really match the desire'd behaviour.

You can write the implementation first (.ml), or the interface first (.mli) as you prefer. If you write the .mli first, trying to compile an implementation that doesn't match the requirement will produce a error, that you will have to fix.

If you write the .ml file first, it's probably easier to proceed iteratively. And the initial version of the interface can be produced w/:

$ ocamlc -i tr_alph.ml > tr_alph.mli

(Be sure to check first there is no a previous .mli, otherwise it will be erased.)

After you can edit this .mli to fix details, and to write the included documentation.
